# Ignore List

<!-- Version: 1.0.0 | Date: 2026-06-30 | Status: Requires review -->

This file lists what AI agents must not modify and what must not be treated as authoritative technical ground truth.

## Generated Artifacts

Do not edit these by hand. They are produced by the build.

- `build/` - assembler output. `build/commander.prg` is regenerated by `build.sh`.
- `example/work.d64` - the example disk image. Regenerated by `example/build-work-d64.sh` and refreshed by `build.sh`. Edit the build script, not the image.

The single source of truth for program behaviour is `src/commander.asm`, not any built artifact.

## Version Control and Tooling

- `.git/` - repository internals.

## Restricted Documentation Directories

These hold historical or on-demand material. Do not read them automatically or treat them as authoritative for current work. Read them only when the user requests the relevant work.

- `docs/change/` - change requests (change history).
- `docs/plan/` - implementation plans.
- `docs/refactoring/` - refactoring proposals and assessments.
- `docs/report/` - on-demand reports.
- `docs/archive/` - retired documents.
- `docs/reference/` - external reference material, consulted only for the topics it covers.

## Submodules

- `docs/skill/commodore-pet-skill/` - the commodore-pet-skill git submodule (upstream: `https://github.com/zoltraks/commodore-pet-skill`). This is the natural location for skills used in this project and is not restricted from reading. Consult it when working on code (see `GUIDELINES.md` "Sources of Truth"). The submodule may be edited directly in this repository; the user commits inside the submodule and synchronizes the pointer. See `README.md` ("Cloning this repository") and `REFERENCES.md`.

## Notes

- Generated artifacts may be inspected for debugging (e.g. checking the PRG load-address bytes), but never hand-edited.
- If a generated file appears out of date, rebuild it with the appropriate script rather than editing it.
